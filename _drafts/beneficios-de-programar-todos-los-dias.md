# Beneficios de programar todos los días

La programación ha sido una de mis pasiones desde muy jóven. Cuando tenía alrededor de 16 años descubrí este mundo del desarrollo de software y desde entonces siempre he estado involucrado de una u otra forma. Desde que entre a la universidad y, posteriormente comencé a trabajar, he estado escribiendo por lo menos algo de código casi todos los días.

<img src="/assets/img/mac-code.jpg" alt="Equilibrio" width="100%"/>

Les mencionaré algunos beneficios de programar con frecuencia.

## Herramientas
<p align="center">
<img src="/assets/img/vscode-logo.png" alt="Visual Studio Code" width="150px" />
</p>

A medida que desarrollamos software para satisfacer distintas necesidades, aprendemos gradualmente nuevas herramientas que nos facilitan el trabajo. En esta búsqueda de opciones que nos faciliten lo que hacemos y reduzcan el tiempo que necesitamos para el mismo, es inevitable encontrar cada vez más y mejores formas de hacer las cosas. 

Entre las herramientas que han acelerado como nunca el desarrollo de aplicaciones para mí se encuentran:
* [Vue](https://vuejs.org/): Framework JavaScript para el desarrollo de aplicaciones web.
* [Vuetify](https://vuetifyjs.com/en/): Decenas de componentes que usan Vuejs para crear interfaces con Material Design.
* [NativeScript](https://www.nativescript.org/): Framework para desarrollar aplicaciones nativas usando JavaScript.
* [Visual Studio Code](https://code.visualstudio.com/): Excelente editor de text (estilo [Sublime Text](https://www.sublimetext.com/)), solo que es casi un IDE pues muestra errores de sintaxis y permite la ejecución y depuración de aplicaciones. _No confundir con Visual Studio._
* [Jekyll](https://jekyllrb.com/): Generador de HTML a partir de texto plano. Excelente para crear documentación o blogs.
* [Firebase](https://firebase.google.com/): Plataforma de Google que provee de bases de datos no relacionales, autenticación, hosting, entre otras herramientas que permiten crear aplicaciones completas sin necesidad de un servidor.

## Buenas prácticas
<img src="/assets/img/code-gp.gif" alt="Buenas prácticas de programación" width="100%" />

Este es un punto muy importante y que muchas veces ignoramos. Escribimos código que instruye al computador a realizar una u otra tarea, sin embargo, en muchos casos lo hacemos sin un orden definido, programamos de acuerdo nos dicta nuestra imaginación y conforme se nos ocurren las ideas. Esto hace que cuando otra persona quiera o necesite leer y entender lo que escribimos, se vuelva muy complicado.

Es esto por lo que programar de forma consistente nos ayuda a entendernos mejor y a los demás. En este artículo "[Escribe código más legible con los siguientes consejos]()" muestro un conjunto de buenas prácticas a la hora de escribir código. Esto con ejemplos en JavaScript y PHP, aunque aplican para cualquier lenguaje.
